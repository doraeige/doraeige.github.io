axios 请求改造封装

发表于 2019-03-16 | 分类于 Vue
字数统计: 724 | 阅读时长 ≈ 3

主要目标功能

  • 快速请求，快速绑定
  • 防止重复提交

模块梳理

  • 接口模块
  • 工具模块
  • 请求模块

架构设计要点

  • 利用 es6 的模块化规范，作为一个封装模块
  • 直接无new 化构建对象
  • 注入vue原型中，让后面的组件可以直接拿到
  • 功能添加采用 原型式与静态结合

实现要点

  • 保证可扩展性，可维护性
  • 常用的方式设为默认加快速度
  • 灵活控制目标data，请求回来直接自动绑定数据

代码演示

static 下新建 json1.json、json2.json文件 模拟一个接口数据
往 Vue 原型链中注入 qa 便于每个子组件都能拿到qa方法

小结

设计思路： 功能确定 > 模块梳理 > 架构基础 > 功能构建
